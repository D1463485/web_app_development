# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述

### 背景與動機
平常在網路上或書本上看到很多不錯的食譜，但經常會忘記存放在哪裡。希望有一個集中管理的地方，可以快速記錄、整理並搜尋所需的食譜與對應食材，讓下廚變得更輕鬆有效率。

### 目標用戶
- 開發者本人（我自己）
- 需要整理散落各地食譜資訊的料理愛好者

### 核心價值主張
提供一個簡單、直覺的個人化食譜管理空間，讓使用者能快速寫下食譜步驟、記錄所需材料，並在需要時透過關鍵字迅速找到想要做的料理。

---

## 2. 功能需求

### 主要功能清單
1. 新增食譜
2. 搜尋食譜
3. 列出材料清單
4. 編輯與刪除食譜
5. 標籤與分類管理

### 使用者故事 (User Stories)
- **新增食譜**：作為**使用者**，我希望**能新增包含名稱、步驟與材料的食譜**，以便**將好吃的料理記錄下來**。
- **搜尋食譜**：作為**使用者**，我希望**能透過關鍵字搜尋食譜名稱**，以便**在不知道要煮什麼的時候能快速找到靈感**。
- **列出材料**：作為**使用者**，我希望**能在單獨的區塊或清單中檢視食譜的材料**，以便**出門採買時能方便核對**。
- **編輯與刪除食譜**：作為**使用者**，我希望**能修改以前寫下的食譜或刪除不再需要的食譜**，以便**維持食譜庫的正確性與整潔**。
- **標籤與分類管理**：作為**使用者**，我希望**能為食譜加上標籤（如：台式、甜點、快速料理）**，以便**更有條理地分類與篩選食譜**。

---

## 3. 非功能需求

### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite

### 效能與安全考量
- **效能**：系統主要為個人使用，資料量不大，應確保頁面載入時間與搜尋反應迅速。
- **安全**：防範基本的 SQL Injection（使用參數化查詢或 ORM）以及 XSS（Jinja2 預設開啟 Autoescape）。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有，第一階段上線核心)
- 新增食譜（包含名稱、作法、材料）
- 瀏覽所有食譜列表（首頁）
- 檢視單一食譜的詳細內容與材料

### Should Have (應該擁有，提升核心體驗)
- 編輯與刪除現有食譜
- 依照名稱搜尋食譜

### Nice to Have (可以擁有，後續擴充亮點)
- 食譜標籤與分類系統
- 材料匯總成購物清單功能（例如勾選已買的材料）
- 食譜圖片上傳
